#  4.12 环境变量

环境变量是所有程序的状态表（但只能读取自己的）
- 它可以告诉程序如何运行：如当环境变量中key=A时，程序以A状态运行
- 可以存储密码
- 可以告诉命令行指令的路径（path）
- 也可以纯当全局变量用

环境变量是一种KeyValue型结构，一个名称对应一个值：key=value
**PS：只有path才会在运行时被终端查找，想让程序在任何路径都可以运行必须将环境变量加到path下**
# env命令（查询环境变量）
```shell
env
```
可以通过管道符+grep过滤
例：过滤含path的环境变量
```shell
env | grep PATH
```
输出：
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
**PATH变量可包含了多个路径，用冒号(:)分隔，系统会在这些路径中依次查找命令**
# $符：取变量的值
环境变量也是变量，所以
$刀乐符的作用是：按键（key）的名字取环境变量中对应的值（value）
如：
```shell
$PATH
#不要这么写
```
但这种会先显示后报错：因为shell展开命令后发现没有这个文件/夹,所以直接返回了展开后的命令
如果path的第一个是一个文件，则这种写法会执行第一个文件

所以想显示环境变量的值应该这么写↓
```shell
echo $PATH
#这是对的
```
或这样↓防止歧义
```shell
echo ${PATH}
#这也是对的
```
# export命令（临时设置环境变量）
```shell
export 键=值
```
**临时配置，终端关闭即失效**
# 配置环境变量（永久生效）
### 仅对当前用户生效
1. 编辑文件 `~/.bashrc` 文件，在最下面添加 `export 键=值`
1. 退出后执行`source 配置文件路径`即可立即生效
或重新打开终端
### 对所以用户生效
1. 编辑文件 `/etc/profile` 文件，在最下面添加`export 键=值`
1. 退出后执行`source 配置文件路径`即可立即生效
或重新打开终端
## 注意⚠️
修改Path环境变量：需要先取值再添加
```shell
export PATH=xxx
#这种会将path原有变量覆盖掉，丢失原有变量
```
```shell
export PATH=$PATH:xxx
#这样不会覆盖（就这么干）
```
