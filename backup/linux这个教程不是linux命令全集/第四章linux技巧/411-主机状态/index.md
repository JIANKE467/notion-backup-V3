#  4.11 主机状态

# 查看系统资源占用top命令
```shell
top
```
- 5秒刷新一次
- q或ctrl加c退出
- 显示CPU，内存使用情况，进程信息
## 详细内容
### 顶部
**第一行**
当前系统时间
系统启动时间
当前用户数
平均负载（3个）1分钟，5分钟，15分钟
[负载是一代表有1个核心100%繁忙，是二就是有2个核心100%繁忙]
**第二行**
task：
total（总进程数），
Running（运行进程数），
Sleeping（睡眠进程数），
Stopped（停止进程数），
zombie（僵尸进程数）
**第三行**
%cpu（s）：
us（用户CPU使用率），
sy（系统CPU使用率），
ni（高优先进程CPU使用率），
id（空闲CPU率），
wa（CPU等待io占用率），
hi（CPU硬件中断率），
si（CPU软件中断率），
st（强制等待占用CPU率）
**第四行**
kib mem：（物理内存）
total（内存总量），
free（内存空闲量），
used（内存使用量），
buff/cache（缓存占用量）
**第五行**
kib swap：（交换分区）（虚拟内存）
与第四行相同
### 进程栏
PID：进程id
USER:进程所属用户
PR:进程优先级,越小越高
M:负值表示高优先级，正表示低优先级VIRT:进程使用虚拟内存，单位KB
RES:进程使用物理内存，单位KB
SHR:进程使用共享内存，单位KB
s:进程状态(S休眠，R运行， Z僵死状态， N负数优先级，I空闲状态)
%CPU:进程占用CPU率
%MEM:进程占用内存率
TIME+:进程使用CPU时间总计，单位10毫秒
COMMAND:进程的命令或名称或程序文径
## top支持选项
| 选项 | 功能 |
| --- | --- |
| -p | 只显示某个进程的信息 |
| -d | 设置刷新时间，默认5s |
| -c | 显示产生进程的完整命令，默认是进程名 |
| -n | 指定刷新次数，达到次数后退出 |
| -b | 以非交互非全屏模式运行，以批次方式执行top，一般配合杠n指定输出几次统计信息，并将输出的重定向到某文件如：top -p -n 3 > /tmp/top.tmp |
| -i | 不显示任何闲置(idle)或无用(zombie)的进程 |
| -u | 查找特定用户启动的进程 |
## top的交互式选项
在top命令运行过程中，可以按一下某些键进行交互
| 按键 | 功能 |
| --- | --- |
| h | 显示帮助画面 |
| c | 显示完整命令 |
| f | 选择需要展示的项目（按下f后用上下左右键调整光标位置，按一下空格可改变可见性状态，Q退出） |
| M | 根据驻留内存大小进行排序 |
| P | 根据CPU使用百分比大小进行排序 |
| T | 根据时间/累计时间进行排序 |
| E | 切换顶部内存显示单位 |
| e | 切换进程内存显示单位 |
| l | 切换显示平均负载和启动时间信息 |
| i | 不显示闲置或无用的进程 |
| t | 切换显示CPU状态信息 |
| m | 切换显示内存信息 |
| 1 | 显示所有CPU核心的负载信息 |
| k | 终止一个进程（需要输入PID） |
# 磁盘信息监控
### df命令（查看磁盘占用）
```shell
df [-h]
```
- -h：人性化显示单位
### iostat命令（CPU占用，磁盘读写速率）
```shell
iostat [-x][num1][num2]
```
- -x：显示更多信息
- num1：数字，刷新间隔
- num2：数字，刷新次数
- 什么选项参数都不加，默认只显示一次
# 网络状态监控
使用sar命令查看网络相关统计（Sar命令非常复杂，这里仅用一种固定的写法）
```shell
sar -n DEV num1 num2
```
- -n DEV：固定写法
- num1：刷新间隔（不填就刷新一次）
- num2：刷新次数（不限制无限刷新）
### 输出信息解读
| 字段 | 含义 |
| --- | --- |
| IFACE | 网络接口名称 |
| rxpck/s | 每秒接收的数据包数量 |
| txpck/s | 每秒发送的数据包数量 |
| rxkB/s | 每秒接收的数据量（KB） |
| txkB/s | 每秒发送的数据量（KB） |
| rxcmp/s | 每秒接收的压缩数据包数 |
| txcmp/s | 每秒发送的压缩数据包数 |
| rxmcst/s | 每秒钟接收的多播数据包 |
