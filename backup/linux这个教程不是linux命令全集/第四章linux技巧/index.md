[4.2 软件安装](42-软件安装/index.md)
  Linux系统支持下载安装包安装，和“软件商店”yum命令安装 两种方式
  ### yum命令（cent OS）
  yum是一个RPM包管理器，可以自动化安装配置Linux软件，可自动寻找依赖
  语法：
  ```shell
yum [-y] [install | remove | update | search | localinstall] 软件包名或路径 
  ```
  - -y：自动确认
  - install：安装
  - remove：卸载
  - update：升级
  - search：搜索
  - localinstall：从本地安装

  **注意：**
  **yum命令需要root权限**
  **软件可以只写软件包名（最新版），也可以写：**`软件名-版本号-发行号`**（指定版本安装）**
  ### apt命令（Ubuntu）
  apt是Ubuntu的包管理器
  语法：
  命令格式一样，只是命令不同
  ```shell
apt [-y] [install | remove | search] 软件名
  ```
  # 注意⚠️
  ### 原理
  yum相当于一个安装器，它默认从系统自带的应用商店下载安装，但也可以给它添加第三方应用商店，这样它就能从第三方下载安装了
  ### 配置第三方仓库
  Q：为什么要用第三方仓库？
  A：因为官方软件仓库更新太慢，而且某些地区连接不上
  Q：推荐什么方法？
  A：能联网最好联网，不然手动安装很麻烦，实在不能联网在内网找一个可以联网的机器建一个软件仓库，不要尝试手动安装（依赖太多，版本还得对上）

  **方法1:**
  有些第三方软件仓库会repo文件或rpm包，直接下载并安装这个包就会自动配置好（自动配置yun源）

  **方法2:**
  1. 找到yum仓库配置文件，路径如下
  ```plain text
/etc/yum.repos.d/
  ```
  每个仓库对应一个.repo文件

  1. 新建一个.repo文件，推荐文件名小写，见文知义
  1. 编辑这个.repo文件，按以下格式
  ```shell
[repoid]          # 仓库唯一ID
name=仓库名称      #仓库显示名称
baseurl=仓库URL   # 仓库地址
enabled=1         # 是否启用仓库
gpgcheck=1    # 是否验证RPM包签名，保证安全
gpgkey=仓库公钥URL  # 仓库公钥URL
priority=10       # 仓库优先级，值越小越优先
protect=1         # 启用保护，防止被覆盖
metadata_expire=6h # 元数据过期时间，控制缓存更新频率
failovermethod=roundrobin # 多镜像时切换策略
  ```
  Q：为什么腾讯云/阿里云镜像都建议直接替换or修改源repo文件
  A：腾讯/阿里害怕你新建配置优先级低于系统自带配置优先级（在repo文件里控制），导致配置不生效，新建配置完全可以，只是要多看一眼优先级

[4.5 日期/时区](45-日期时区/index.md)
  # date命令（查询系统时间）
    ```shell
date [-d] [加减参数] [+格式化字符串]
    ```
    - -d：在当前时间上进行加减
    - 不带任何参数选项，会显示当前年，月，日，星期，时，分，秒，时区
    - 格式化字符串可以组合使用，可以与字符混合使用
    ### **加减参数：**
    +/-与
    year 年
    month 月
    day 天
    hour 小时
    minute 分钟
    second 秒
    组合使用
    如：（加1月，减3天）
    ```shell
date -d "+1 month -3 days"
    ```
    **如果一次性同时加减多个时间单位，如同时加减月和天，要用双引号引起来，避免shell在空格处断句**

    ### **格式化字符串：**

    | 格式化字符 | 含义 |
    | --- | --- |
    | %Y | 完整年份（如：2026） |
    | %y | 年份后两位（如：26） |
    | %m | 月份（01-12） |
    | %d | 日期（01-31） |
    | %H | 小时（00-23） |
    | %M | 分钟（00-59） |
    | %S | 秒（00-59） |
    | %s | 时间戳：从1970-01-01 00:00:00 UTC到现在的秒数 |
    例子：
    1:按照2026-01-01 的格式显示
    ```shell
date +%Y-%m-%d
    ```
    2:按照2022-01-01 10:00:00 的格式显示
    ```shell
date "+%Y-%m-%d %H:%M:%S＂
    ```
    PS：这里必须带双引号，因为格式化字符串中有空格，shell会从空格处断开
    **双引号：强制识别为命令和字符的混合（将引号内作为一个整体处理，防止shell从空格处断开）**
    **单引号：强制识别为字符串**
    **反引号：强制识别为命令**

[4.7 VMware配置固定IP](47-vmware配置固定ip/index.md)
  ### 为什么需要配置固定IP？
  虚拟机内的IP地址，是虚拟机的DHCP服务分配的
  DHCP：动态IP分配，重启会重新分配
  连接到虚拟机IP地址老变会很麻烦
  ### 步骤
  1. VMware→编辑→虚拟网络编辑器→VMnet8→记住下面的子网IP和子网掩码→nat设置→记住网关IP
  1. 在Linux中编辑文件 `/etc/sysconfig/network-scripts/ifcfg-ens33 `
  1. 修改以下条目
  BOOTPROTO=”dhcp”
  将dhcp改为static

  1. 新增以下条目
  IPADDR="[192.168.xxx.xxx](http://192.168.xxx.xxx/)"（本机IP）（根据子网IP范围设置）
  NETMASK="255.255.255.0"（子网掩码）
  GATEWAY="[192.168.xxx](http://192.168.xxx/).2"（网关IP）
  DNS1="[192.168.xxx](http://192.168.xxx/).2"（DNS服务器，可设置为网关IP）

  1. 重启网卡服务
  命令：
  `systemctl restart network`
  或：`ifdown ens33 && ifup ens33`（针对具体网卡）

  # 查看系统资源占用top命令
  ```shell
top
  ```
  - 5秒刷新一次
  - q或ctrl加c退出
  - 显示CPU，内存使用情况，进程信息
  ## 详细内容
  ### 顶部
  **第一行**
  当前系统时间
  系统启动时间
  当前用户数
  平均负载（3个）1分钟，5分钟，15分钟
  [负载是一代表有1个核心100%繁忙，是二就是有2个核心100%繁忙]
  **第二行**
  task：
  total（总进程数），
  Running（运行进程数），
  Sleeping（睡眠进程数），
  Stopped（停止进程数），
  zombie（僵尸进程数）
  **第三行**
  %cpu（s）：
  us（用户CPU使用率），
  sy（系统CPU使用率），
  ni（高优先进程CPU使用率），
  id（空闲CPU率），
  wa（CPU等待io占用率），
  hi（CPU硬件中断率），
  si（CPU软件中断率），
  st（强制等待占用CPU率）
  **第四行**
  kib mem：（物理内存）
  total（内存总量），
  free（内存空闲量），
  used（内存使用量），
  buff/cache（缓存占用量）
  **第五行**
  kib swap：（交换分区）（虚拟内存）
  与第四行相同
  ### 进程栏
  PID：进程id
  USER:进程所属用户
  PR:进程优先级,越小越高
  M:负值表示高优先级，正表示低优先级VIRT:进程使用虚拟内存，单位KB
  RES:进程使用物理内存，单位KB
  SHR:进程使用共享内存，单位KB
  s:进程状态(S休眠，R运行， Z僵死状态， N负数优先级，I空闲状态)
  %CPU:进程占用CPU率
  %MEM:进程占用内存率
  TIME+:进程使用CPU时间总计，单位10毫秒
  COMMAND:进程的命令或名称或程序文径

[4.12 环境变量](412-环境变量/index.md)
  环境变量是所有程序的状态表（但只能读取自己的）
  - 它可以告诉程序如何运行：如当环境变量中key=A时，程序以A状态运行
  - 可以存储密码
  - 可以告诉命令行指令的路径（path）
  - 也可以纯当全局变量用

  环境变量是一种KeyValue型结构，一个名称对应一个值：key=value
  **PS：只有path才会在运行时被终端查找，想让程序在任何路径都可以运行必须将环境变量加到path下**
  # env命令（查询环境变量）
  ```shell
env
  ```
  可以通过管道符+grep过滤
  例：过滤含path的环境变量
  ```shell
env | grep PATH
  ```
  输出：
  PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
  **PATH变量可包含了多个路径，用冒号(:)分隔，系统会在这些路径中依次查找命令**
  # $符：取变量的值
  环境变量也是变量，所以
  $刀乐符的作用是：按键（key）的名字取环境变量中对应的值（value）
  如：
  ```shell
$PATH
#不要这么写
  ```
  但这种会先显示后报错：因为shell展开命令后发现没有这个文件/夹,所以直接返回了展开后的命令
  如果path的第一个是一个文件，则这种写法会执行第一个文件

  所以想显示环境变量的值应该这么写↓
  ```shell
echo $PATH
#这是对的
  ```
  或这样↓防止歧义
  ```shell
echo ${PATH}
#这也是对的
  ```

[4.14 压缩解压](414-压缩解压/index.md)
  ### 压缩包格式：
  zip：Linux，win，mac常用
  7zip：win
  rar：win
  tar：Linux，mac常用
  gzip：Linux，mac常用
  ### Linux中常见压缩文件后缀
  `.tar`：归档文件，只是简单将文件封装到.tar文件内，压缩率不高
  `.gz`：是gzip格式的压缩文件，压缩率较高
  # tar/gz包命令：tar（压缩解压）
  tar命令对.tar和.gz均有效
  **语法：**
  ```shell
tar [-c -v -x -f -z -C] 参数1 参数2...
  ```
  - -c：创建压缩文件
  - -v：显示进度
  - -x：解压
  - -f：指定压缩包路径（含文件名）
  -f选项必须在选项位的最后一个，不使用-f则默认直接输出压缩文件到终端，-f后的第一个路径就是指定路径
  - -z：压缩成.gz，不选默认是.tar包
  - -C：选择解压路径
  （建议单独使用，和其他参数分开，避免歧义）
  - 参数1：如果有-f则这个是压缩文件指定路径，否则就是要被压缩的文件
  - 参数2~n：要被压缩的文件路径 
  ps：
  1. .gz解压时要带-z选项，不能只带-x
  1. -f放到最后
  1. -C建议单独使用↓
  ```shell
tar -zxvf test.rar -C /home
  ```
  # zip压缩包命令：zip
  ### 压缩
  ```shell
zip [-r] 参数1 参数2...
  ```
  - -r：被压缩的包含文件夹
  -  参数1：压缩包路径
  - 参数2~n：被压缩文件路径
  ### 解压
  ```shell
unzip 压缩包路径 [-d] 路径
  ```
  - -d：指定解压路径
  PS：解压时有同名内容会被直接替换
