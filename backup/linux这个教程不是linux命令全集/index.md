    # find命令（查找文件）
    find命令用来查找文件
    ### 语法（按文件名查找）
    ```shell
find 起始路径 -name "被查找文件名"
    ```
    PS：
    起始路径指定了从哪搜索
    find支持通配符模糊匹配
    ### 语法（按大小查找）
    ```shell
find 起始路径 -size +|-n[kMG]
    ```
    PS：
    - +|-：代表大于或小于
    - n：代表大小
    - kMG：代表单位：k（小写）表示kb，M表示MB，G表示GB
    例子：
    在根目录查找小于10kb的文件
    ```shell
find / -size -10k
    ```
    如果是区间限制（如10k到10M），这么写
    ```shell
find / -size +10k -size -10M
    ```

  3.1 认识root用户
    root用户是超级管理员，有最大权限

    # su命令（切换用户）
    switch user 的简写，切换用户
    ### 语法
    ```shell
su [-] [用户名]
    ```
    - -符号：表示是否在切换后加载环境变量
    - 用户名：省略默认切换到root用户
    - 使用`exit命令`退出到上一个用户
    - root切换到另一个用户不用输密码
    # sudo命令（临时使用root权限）
    ### 语法
    ```shell
sudo 其他命令
    ```
    - **只有配置过sudo认证的用户才能用sudo命令**
    ### 配置sudo认证
    1. 切换到root用户
    1. 执行visudo命令，会用编辑器自动打开/etc/sudoers
    1. 在文件最后添加
    ```plain text
用户名 ALL=(ALL)    NOPASWARD: ALL
    ```
    NOPASSWARD：ALL 表示使用sudo命令无需输密码
    1. 保存，退出root用户

    **部分系统的初始用户没配置sudo也能用sudo是因为，系统在安装时自动将初始用户加入了sudo组**

  3.3 查看权限控制
    本节没有命令，只是阐述Linux权限
    在 ls -l 中，结果以：
    **权限信息**，**硬连接数**，**所属用户**，**所属用户组**，**文件大小**，**最后修改时间**，**文件名**
    来展示
    # 权限信息
    权限信息用10位的字符表示：
    ![](assets/3155a2dd-8276-8155-8bb5-f5f778c1d927.png)
    **除了第一位外，剩下的9位被分为三组，分别是：****所属用户权限****，****所属用户组权限****，****其他用户权限**
    ### 第一位：-/d/l
    -：表示文件
    d：表示文件夹
    l：表示软链接
    ### 第2~4位：（所属用户权限）
    三位按先后顺序分别是r w x
    r/-：read ，有/无读权限
    w/-：write，有/无写权限
    x/-：execute，有/无执行权限
    ### 第5~7位：（所属用户组权限）
    同样是r w x三位，表示文件所属用户组的读、写、执行权限
    ### 第8~10位:(其他用户权限)
    同样是r w x三位,表示其他用户的读、写、执行权限
    ### 解释
    读权限：只能查看（如ls）
    写权限：增删改
    执行：运行（如cd）
  3.4 修改权限控制
    ## chmod命令（修改权限）（文件所属用户/root用户才可用）
    ```shell
chmod [-R] 权限 文件或文件夹
    ```
    - -R：递归修改
    PS：权限有两种书写格式，数字格式/字母格式
    ### 权限表示（数字格式）：
    ```shell
chmod [-R] xxx 文件或文件夹
    ```
    数字模式的三位分别表示：**所属用户、所属组、其他用户**的权限
    每位的数字是对应权限之和（权限数字如下）
    | 权限 | 数字 |
    | --- | --- |
    | r（读） | 4 |
    | w（写） | 2 |
    | x（执行） | 1 |
    ### 权限表示（字母格式）：
    ```shell
chmod [-R] u=rwx g=rwx o=rwx 文件或文件夹
    ```
    其中：**u**表示所属用户（user），**g**表示所属组（group），**o**表示其他用户（others），**a**表示所有人（all，相当于ugo）
    可以使用`+`、`-`、`=`来添加、删除、设置权限，例如：
    - `chmod u+x,g-w,o=r 文件名`
    - `chmod a+x 文件名`（给所有人添加执行权限，等同于 `chmod u+x,g+x,o+x 文件名`）
    ## chown命令（修改所属用户/组）（仅root权限）
    ```shell
chown [-R] [用户][:][用户组] 文件或文件夹
    ```
    选项说明：
    - -R：递归修改
    - 冒号：用于分隔用户和用户组
    PS：
    只改用户组，需要带冒号

    ### 配置第三方仓库
    Q：为什么要用第三方仓库？
    A：因为官方软件仓库更新太慢，而且某些地区连接不上
    Q：推荐什么方法？
    A：能联网最好联网，不然手动安装很麻烦，实在不能联网在内网找一个可以联网的机器建一个软件仓库，不要尝试手动安装（依赖太多，版本还得对上）

    **方法1:**
    有些第三方软件仓库会repo文件或rpm包，直接下载并安装这个包就会自动配置好（自动配置yun源）

    **方法2:**
    1. 找到yum仓库配置文件，路径如下
    ```plain text
/etc/yum.repos.d/
    ```
    每个仓库对应一个.repo文件

    1. 新建一个.repo文件，推荐文件名小写，见文知义
    1. 编辑这个.repo文件，按以下格式
    ```shell
[repoid]          # 仓库唯一ID
name=仓库名称      #仓库显示名称
baseurl=仓库URL   # 仓库地址
enabled=1         # 是否启用仓库
gpgcheck=1    # 是否验证RPM包签名，保证安全
gpgkey=仓库公钥URL  # 仓库公钥URL
priority=10       # 仓库优先级，值越小越优先
protect=1         # 启用保护，防止被覆盖
metadata_expire=6h # 元数据过期时间，控制缓存更新频率
failovermethod=roundrobin # 多镜像时切换策略
    ```
    Q：为什么腾讯云/阿里云镜像都建议直接替换or修改源repo文件
    A：腾讯/阿里害怕你新建配置优先级低于系统自带配置优先级（在repo文件里控制），导致配置不生效，新建配置完全可以，只是要多看一眼优先级

    # date命令（查询系统时间）
      ```shell
date [-d] [加减参数] [+格式化字符串]
      ```
      - -d：在当前时间上进行加减
      - 不带任何参数选项，会显示当前年，月，日，星期，时，分，秒，时区
      - 格式化字符串可以组合使用，可以与字符混合使用
      ### **加减参数：**
      +/-与
      year 年
      month 月
      day 天
      hour 小时
      minute 分钟
      second 秒
      组合使用
      如：（加1月，减3天）
      ```shell
date -d "+1 month -3 days"
      ```
      **如果一次性同时加减多个时间单位，如同时加减月和天，要用双引号引起来，避免shell在空格处断句**

      ### **格式化字符串：**

      | 格式化字符 | 含义 |
      | --- | --- |
      | %Y | 完整年份（如：2026） |
      | %y | 年份后两位（如：26） |
      | %m | 月份（01-12） |
      | %d | 日期（01-31） |
      | %H | 小时（00-23） |
      | %M | 分钟（00-59） |
      | %S | 秒（00-59） |
      | %s | 时间戳：从1970-01-01 00:00:00 UTC到现在的秒数 |
      例子：
      1:按照2026-01-01 的格式显示
      ```shell
date +%Y-%m-%d
      ```
      2:按照2022-01-01 10:00:00 的格式显示
      ```shell
date "+%Y-%m-%d %H:%M:%S＂
      ```
      PS：这里必须带双引号，因为格式化字符串中有空格，shell会从空格处断开
      **双引号：强制识别为命令和字符的混合（将引号内作为一个整体处理，防止shell从空格处断开）**
      **单引号：强制识别为字符串**
      **反引号：强制识别为命令**

  4.7 VMware配置固定IP
    ### 为什么需要配置固定IP？
    虚拟机内的IP地址，是虚拟机的DHCP服务分配的
    DHCP：动态IP分配，重启会重新分配
    连接到虚拟机IP地址老变会很麻烦
    ### 步骤
    1. VMware→编辑→虚拟网络编辑器→VMnet8→记住下面的子网IP和子网掩码→nat设置→记住网关IP
    1. 在Linux中编辑文件 `/etc/sysconfig/network-scripts/ifcfg-ens33 `
    1. 修改以下条目
    BOOTPROTO=”dhcp”
    将dhcp改为static

    1. 新增以下条目
    IPADDR="[192.168.xxx.xxx](http://192.168.xxx.xxx/)"（本机IP）（根据子网IP范围设置）
    NETMASK="255.255.255.0"（子网掩码）
    GATEWAY="[192.168.xxx](http://192.168.xxx/).2"（网关IP）
    DNS1="[192.168.xxx](http://192.168.xxx/).2"（DNS服务器，可设置为网关IP）

    1. 重启网卡服务
    命令：
    `systemctl restart network`
    或：`ifdown ens33 && ifup ens33`（针对具体网卡）

  4.12 环境变量
    环境变量是所有程序的状态表（但只能读取自己的）
    - 它可以告诉程序如何运行：如当环境变量中key=A时，程序以A状态运行
    - 可以存储密码
    - 可以告诉命令行指令的路径（path）
    - 也可以纯当全局变量用

    环境变量是一种KeyValue型结构，一个名称对应一个值：key=value
    **PS：只有path才会在运行时被终端查找，想让程序在任何路径都可以运行必须将环境变量加到path下**
    # env命令（查询环境变量）
    ```shell
env
    ```
    可以通过管道符+grep过滤
    例：过滤含path的环境变量
    ```shell
env | grep PATH
    ```
    输出：
    PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
    **PATH变量可包含了多个路径，用冒号(:)分隔，系统会在这些路径中依次查找命令**
    # $符：取变量的值
    环境变量也是变量，所以
    $刀乐符的作用是：按键（key）的名字取环境变量中对应的值（value）
    如：
    ```shell
$PATH
#不要这么写
    ```
    但这种会先显示后报错：因为shell展开命令后发现没有这个文件/夹,所以直接返回了展开后的命令
    如果path的第一个是一个文件，则这种写法会执行第一个文件

    所以想显示环境变量的值应该这么写↓
    ```shell
echo $PATH
#这是对的
    ```
    或这样↓防止歧义
    ```shell
echo ${PATH}
#这也是对的
    ```

  4.14 压缩解压
    ### 压缩包格式：
    zip：Linux，win，mac常用
    7zip：win
    rar：win
    tar：Linux，mac常用
    gzip：Linux，mac常用
    ### Linux中常见压缩文件后缀
    `.tar`：归档文件，只是简单将文件封装到.tar文件内，压缩率不高
    `.gz`：是gzip格式的压缩文件，压缩率较高
    # tar/gz包命令：tar（压缩解压）
    tar命令对.tar和.gz均有效
    **语法：**
    ```shell
tar [-c -v -x -f -z -C] 参数1 参数2...
    ```
    - -c：创建压缩文件
    - -v：显示进度
    - -x：解压
    - -f：指定压缩包路径（含文件名）
    -f选项必须在选项位的最后一个，不使用-f则默认直接输出压缩文件到终端，-f后的第一个路径就是指定路径
    - -z：压缩成.gz，不选默认是.tar包
    - -C：选择解压路径
    （建议单独使用，和其他参数分开，避免歧义）
    - 参数1：如果有-f则这个是压缩文件指定路径，否则就是要被压缩的文件
    - 参数2~n：要被压缩的文件路径 
    ps：
    1. .gz解压时要带-z选项，不能只带-x
    1. -f放到最后
    1. -C建议单独使用↓
    ```shell
tar -zxvf test.rar -C /home
    ```
    # zip压缩包命令：zip
    ### 压缩
    ```shell
zip [-r] 参数1 参数2...
    ```
    - -r：被压缩的包含文件夹
    -  参数1：压缩包路径
    - 参数2~n：被压缩文件路径
    ### 解压
    ```shell
unzip 压缩包路径 [-d] 路径
    ```
    - -d：指定解压路径
    PS：解压时有同名内容会被直接替换
